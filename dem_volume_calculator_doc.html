DEM Volume Calculator

Plugin QGIS 3 per il calcolo del volume di scavo o riporto dalla differenza tra due superfici DEM generate da punti quotati. Supporta interpolazione IDW, TIN e Kriging, genera report PDF certificati con testo AI e verifica l'integrità dei dati tramite hash SHA-256.

📌 QGIS ≥ 3.28 🐍 Python 3.8+ 👤 Luca Mandolesi 🔖 v1.3.0 📂 Categoria: Raster 🏛️ pyArchInit
⚱️
Parte dell'ecosistema pyArchInit — Gestione dati archeologici con QGIS
pyArchInit è un framework open source per la gestione integrata dei dati di scavo archaeologico su piattaforma GIS. Nato nel 2005, consente di documentare unità stratigrafiche, materiali, campioni e georeferenziare ogni dato direttamente in QGIS. Il DEM Volume Calculator si integra nell'ecosistema per calcolare volumi di scavo e riporto a partire dai punti quotati rilevati sul campo, producendo report tecnici certificati direttamente utilizzabili nella documentazione di scavo.
→ pyarchinit.org

CSV (auto-detect separatore), Shapefile, GeoTIFF, ASC, GRD, IMG, FLT, BIL, DEM, HGT, NetCDF e tutti i formati GDAL.

📦Installazione

Metodo 1 – da ZIP (consigliato)

  1. Scarica il file ZIP Dalla pagina GitHub scarica dem_volume_calculator.zip dalla sezione Releases oppure tramite Code → Download ZIP.
  2. Installa da ZIP in QGIS In QGIS: Plugin → Gestisci e installa plugin → Installa da ZIP, seleziona il file e clicca Installa plugin.
  3. Attiva il plugin Nella scheda Installati, abilita la spunta accanto a DEM Volume Calculator. L'icona Δ compare nella barra strumenti Raster.
  4. Installazione automatica dei moduli Al primo avvio, il plugin controlla ReportLab. Se mancante, mostra un dialog per installarla automaticamente con pip (con progress bar). Riavviare QGIS dopo l'installazione.

Metodo 2 – copia manuale della cartella

Copia la cartella dem_volume_calculator/ nella directory plugin di QGIS:

📐Tab Calcoli

Il tab principale è suddiviso in quattro sezioni: Input Dati, DEM Raster Pre-esistenti, Opzioni Raster e Azioni.

Sezione — Input Dati

Richiede due set di punti quotati per definire le due superfici:

  • DEM UP — superficie superiore (es. piano di campagna originale, quota ante-scavo).
  • DEM DOWN — superficie inferiore (es. fondo dello scavo, quota post-scavo).
ModalitàCome si usaFormati
File su disco Pulsante "Carica file quota SUPERIORE/INFERIORE" .csv, .shp
Layer Panel Pulsante "Da Layer Panel" — usa il layer vettoriale puntuale attivo in QGIS Layer vettoriale Point / PointZ

Dopo il caricamento il plugin rileva automaticamente i campi disponibili e popola il menu Campo quota. Il campo Z viene pre-selezionato automaticamente cercando i nomi: Z, QUOTA, ELEVATION, ELEV, HEIGHT, ALT (e le varianti in minuscolo). Per CSV senza intestazione, il default è Colonna_3 (terza colonna).

💡 CSV — auto-detect separatore e intestazione Il plugin analizza i primi 4 KB del file per rilevare il separatore (virgola, punto e virgola, tabulazione). Se la prima riga contiene solo valori numerici, viene trattata come dato (nessuna intestazione) e i campi denominati Colonna_1, Colonna_2, … Colonna_N.

Sezione — DEM Raster Pre-esistenti

Se si dispone già di file raster DEM, è possibile caricarli direttamente saltando il passo 1.

SorgenteAzione
File raster su discoPulsante "Carica file…" per DEM UP o DEM DOWN
Layer raster nel PanelPulsante "Da Layer Panel" — usa il layer raster attivo

Formati raster supportati: .tif/.tiff (GeoTIFF), .asc (ESRI ASCII Grid), .grd (Surfer Grid), .img (ERDAS Imagine), .flt/.bil (ESRI Float/BIL), .dem, .hgt (SRTM HGT), .nc (NetCDF) e tutti i formati GDAL.

Al caricamento il plugin esegue una validazione automatica (validate_raster_dem()): controlla la presenza di almeno una banda, il tipo di dato e l'assenza di errori GDAL. Gli avvisi (es. nodata non impostato) vengono mostrati nel log senza bloccare il caricamento.

ℹ️ Passo 1 saltato automaticamente Se entrambi i DEM raster sono caricati, il pulsante "1. Genera DEM" viene ignorato e si può procedere direttamente con il passo 2 (Differenza) o "Esegui Tutto".

Sezione — Opzioni Raster

ParametroDefaultDescrizione

Dimensione cella DEM DOWN 1.0 m Risoluzione spaziale del raster superficie inferiore.

Sezione — Azioni

  • Passo 1 — Genera DEM UP e DEM DOWN Interpola i punti quotati e produce due GeoTIFF float32 in cartella temporanea (dem_volume_calc_XXXX/dem_up.tif, dem_down.tif). Eseguito in un QThread separato; la barra di avanzamento mostra 10% → 50% → 100%. Dopo l'interpolazione applica la maschera convex hull dei punti di input (celle fuori dall'inviluppo convesso → NoData). Al termine i raster sono caricati nel Layer Panel.
  • Passo 2 — Calcola Differenza Calcola DEM UP − DEM DOWN (o inverso) pixel per pixel. Se i due DEM hanno risoluzioni o estensioni diverse, vengono ricampionati (gdal.Warp, algoritmo bilineare) all'extent comune e alla risoluzione target. Il raster differenza (dem_diff.tif) viene caricato nel Layer Panel.
  • Passo 3 — Calcola Volume Somma i valori del raster differenza (opzionalmente solo i pixel positivi) e moltiplica per l'area del pixel (cell_size²). Il log mostra: volume in m³, statistiche (min/max/media/std), numero di pixel validi, hash SHA-256 dei file DEM.

⚡ Esegui Tutto (1 + 2 + 3)

Concatena automaticamente i tre passi in un'unica operazione. Ogni passo viene registrato con timestamp in _steps_log e incluso nella sezione Metodologia del report PDF.

💡 Opzioni risultati (spunte in fondo al tab) Aggiungi raster risultanti al progetto QGIS (default: ✔) — carica i raster nel Layer Panel al termine.
Considera solo valori positivi per il volume (default: ✔) — ignora i pixel negativi nel calcolo (aree dove DOWN > UP per errori di rilievo o piccole disomogeneità).

📄Tab Report

Il tab Report genera un documento PDF tecnico certificato con:

  • Screenshot cartografici di DEM UP e DEM DOWN con sfondo OpenStreetMap (1600×1200 px, 150 DPI, ricampionamento bilineare).

  • Hash SHA-256 di ciascun DEM per la tracciabilità del dato.

  • Testo interpretativo generato da un modello AI (Ollama, OpenAI o Anthropic).

Ollama — AI locale, gratuito, privacy-safe

I dati non lasciano il computer. Nessuna API key richiesta.

  • Scarica e installa Ollama da ollama.com (Windows, macOS, Linux).
  • Auto-discovery multi-porta: il plugin cerca Ollama su localhost e 127.0.0.1 sulle porte 11434, 11435, 11436, 8080, 8081. Usa la variabile d'ambiente OLLAMA_HOST per configurare un host personalizzato.
  • Modello consigliato: llama3.2:latest. Scaricabile direttamente dal plugin.
  • Il download del modello usa streaming JSON con aggiornamento bytes in tempo reale e pulsante Annulla.
  • Evita modelli molto piccoli (es. gemma3:1b, tinyllama) che potrebbero restituire errore 500.
💡 Download modello dal plugin Seleziona Ollama come provider, scegli il modello dall'elenco e clicca "Scarica modello". Un dialog mostra il terminale con il progresso in bytes e la barra percentuale. Il download gira in OllamaPullWorker (QThread separato) e può essere annullato.

OpenAI — richiede API key e connessione internet

  • Ottieni la key su platform.openai.com.
  • La key viene salvata in ~/.dem_volume_calc/api_keys.json al termine della digitazione (segnalato con un dialog che offre di aprire la cartella).
  • Modelli disponibili: gpt-4o, gpt-4o-mini, gpt-4-turbo, gpt-4.1, gpt-4.1-mini, gpt-4.1-nano, o3, o3-mini, o4-mini.
  • Supporto immagini (visione): screenshot delle mappe inviati come data:image/png;base64 nei modelli vision.
  • In caso di HTTP 429: 3 tentativi con backoff esponenziale (30 s → 60 s → 120 s), rispettando l'header Retry-After se presente.

Anthropic Claude — richiede API key e connessione internet

  • Ottieni la key su console.anthropic.com.
  • La key viene salvata in ~/.dem_volume_calc/api_keys.json (stesso file di OpenAI).
  • Modelli disponibili: claude-sonnet-4-20250514, claude-opus-4-20250514, claude-haiku-4-5-20251001.
  • Header API richiesto: anthropic-version: 2023-06-01 — incluso automaticamente.
  • Supporto immagini: le mappe DEM vengono incluse come blocchi image/base64 nel messaggio user.
  • Stesso meccanismo di retry con backoff esponenziale per HTTP 429.

Struttura del report PDF

Il PDF (formato A4, font Cambria su Windows o Times-Roman su altri sistemi) è strutturato in cinque capitoli:

  1. Copertina — titolo, autore, data, logo aziendale, firma con altezza configurabile.
  2. Introduzione — testo AI (max ~800 parole) che descrive il contesto del calcolo volume in italiano tecnico.
  3. Metodologia — algoritmo di interpolazione con formula matematica dettagliata, parametri GDAL usati, hash SHA-256 dei DEM, log dei passaggi operativi con timestamp.
  4. Risultati — volume in m³ (evidenziato graficamente), statistiche raster (min/max/media/std/pixel validi), screenshot DEM UP e DEM DOWN a sfondo OSM (1600×1200 px, 150 DPI).
  5. Conclusioni — testo AI con interpretazione tecnica del risultato, applicabilità nel computo metrico di scavo/riporto.
ℹ️ Nome file PDF Il PDF viene salvato come report_volume_YYYYMMDD_HHMMSS.pdf nella cartella output selezionata. L'ultimo PDF generato viene ricordato dal plugin per essere incluso nella funzione Salva elaborazione.

📁File di output

Raster GeoTIFF

Tutti i raster prodotti sono in formato GeoTIFF float32 con:

  • CRS nel sistema di riferimento EPSG impostato dall'utente.
  • Valore NoData: -9999.0.
  • Maschera convex hull: le celle fuori dall'inviluppo convesso dei punti di input sono NoData.
  • Buffer automatico dell'extent del 5% per evitare troncatura ai bordi.

Statistiche nel log

Al termine del passo 3, il log riporta:

  • Volume totale in m³ (con indicazione se solo valori positivi o tutti).
  • Statistiche del raster differenza: min, max, media, deviazione standard.
  • Numero pixel validi (non-NoData) usati nel calcolo.
  • Hash SHA-256 di dem_up.tif e dem_down.tif.

📊Metodi di interpolazione

IDW — Inverse Distance Weighting

Media pesata delle quote dei punti circostanti con peso inversamente proporzionale alla distanza (esponente: p = 2.0).

  • Implementazione: gdal.Grid() con algoritmo invdist:power=2.0:smoothing=0.0:radius1=MAX_EXTENT.
  • Post-processing: maschera convex hull tramite OGR + GDAL rasterize.
  • Vantaggi: deterministico, veloce, nessuna dipendenza extra.
  • Svantaggi: "bull's-eye" attorno ai punti di controllo.
  • Ideale per: nuvole di punti dense e uniformi.

TIN — Triangulated Irregular Network (Delaunay)

Triangolazione di Delaunay + interpolazione lineare all'interno di ogni triangolo.

  • Implementazione: gdal.Grid() con algoritmo linear:radius=MAX_EXTENT:nodata=-9999.
  • Post-processing: maschera convex hull applicata dopo la griglia.
  • Vantaggi: rispetta esattamente i valori misurati; lineare a tratti.
  • Svantaggi: discontinuità agli angoli dei triangoli; aree piatte senza punti.
  • Ideale per: rilievi con distribuzione irregolare ma morfologicamente rappresentativa.

Kriging — RBFInterpolator (thin_plate_spline)

Interpolazione con funzioni base radiali (RBF) tramite kernel thin plate spline.

  • Implementazione: scipy.interpolate.RBFInterpolator(kernel='thin_plate_spline') → output su griglia numpy → gdal.GetDriverByName('GTiff').Create().
  • Fallback automatico: se SciPy non è disponibile, usa IDW e lo segnala nel log.
  • Post-processing: maschera convex hull applicata.
  • Vantaggi: superficie più fluida; ottimo su dataset sparsi o irregolari.
  • Svantaggi: più lento; richiede SciPy.
  • Ideale per: superfici continue con pochi punti di rilievo.
⚠️ Dipendenza SciPy Senza SciPy: fallback automatico a IDW. Il log riporta: "Scipy non disponibile — fallback a IDW". Installare con: pip install scipy.

🩺Diagnostica Ollama

Il modulo ollama_checker.py esegue una diagnostica strutturata in 6 check e restituisce un oggetto OllamaDiagnostics con report testuale.

  • 1
    Binario Ollama nel PATH Usa shutil.which('ollama'). Se trovato, esegue ollama --version per leggere la versione.
  • 2
    Processo Ollama in esecuzione Windows: tasklist /FI IMAGENAME eq ollama.exe. Linux/macOS: pgrep -x ollama.
  • 3
    Variabile d'ambiente OLLAMA_HOST Legge $OLLAMA_HOST. Se impostata, viene usata come URL preferenziale (accetta formato host:port o URL completo).
  • 4
    Connettività API Ollama Controlla le porte 11434, 11435, 11436, 8080, 8081 su localhost e 127.0.0.1. Verifica risposta all'endpoint /api/version (fallback: endpoint root).
  • 5
    Modelli installati Interroga /api/tags. Riporta nome e numero dei modelli. Se nessun modello: suggerisce ollama pull llama3.2:latest.
  • 6
    Rilevamento GPU NVIDIA: nvidia-smi --query-gpu=name,memory.total. Windows: wmic path win32_VideoController. Linux: lspci. Non bloccante: senza GPU Ollama usa la CPU.

📥Installazione automatica moduli (dep_checker)

Il modulo dep_checker.py gestisce il rilevamento e l'installazione automatica dei moduli Python opzionali direttamente dall'interfaccia QGIS.

Funzionamento

  1. All'avvio del plugin, log_missing() segnala i moduli mancanti nel message bar e nel log QGIS (non bloccante).
  2. Al primo click su una funzione che richiede un modulo mancante, check_and_prompt() mostra un dialog con l'elenco dei moduli da installare e i comandi pip alternativi.
  3. Se l'utente accetta, avvia l'installazione con una QProgressDialog (barra indeterminata).
  4. L'installazione prova prima pip install --user, poi senza flag se fallisce.
  5. Al termine mostra un dialog di conferma e avvisa di riavviare QGIS per rendere disponibili i moduli.
ℹ️ Modulo attualmente gestito Il dep_checker gestisce attualmente ReportLab (per la generazione PDF). SciPy non è incluso nel check automatico: il plugin gestisce la sua assenza con un fallback diretto a IDW senza interrompere il flusso.

🔍Risoluzione problemi

❌ "ReportLab non trovato" — PDF non generato

Installare ReportLab via dep_checker (avvia il plugin e conferma nel dialog) o manualmente:

import subprocess, sys
subprocess.check_call([sys.executable, '-m', 'pip', 'install', 'reportlab'])

❌ Kriging usa IDW (fallback automatico)

SciPy non è installato. Il plugin usa IDW automaticamente. Per abilitare Kriging:

import subprocess, sys
subprocess.check_call([sys.executable, '-m', 'pip', 'install', 'scipy'])

❌ Ollama non raggiungibile

Verificare che il servizio Ollama sia in esecuzione:

# Avvia Ollama (Linux / macOS)
ollama serve

# Verifica connettività
curl http://localhost:11434/api/version

Se Ollama gira su un host diverso, impostare OLLAMA_HOST=http://<ip>:11434 prima di avviare QGIS. Usare il tool di diagnostica per un report completo (vedi Diagnostica Ollama).

❌ Errore Ollama 404 — modello non trovato

Il modello selezionato non è installato localmente. Scaricarlo con: ollama pull <nome_modello> oppure tramite il pulsante "Scarica modello" nel tab Report.

❌ Errore Ollama 500 — Internal Server Error

Cause probabili:

  • Modello troppo piccolo per il prompt (es. gemma3:1b, tinyllama) — usare llama3.2, gemma3:4b, mistral, qwen2.5.
  • RAM/VRAM insufficiente — chiudere altre applicazioni prima di generare il report.
  • Corruzione del modello — riscaricare con ollama pull <modello>.

❌ Errore 429 — rate limit API (OpenAI / Anthropic)

Il plugin esegue automaticamente 3 tentativi con backoff esponenziale (30 s → 60 s → 120 s, rispettando l'header Retry-After). Se persiste, verificare il saldo/crediti sul portale del provider.

❌ Il volume risulta 0 o negativo

  • Controllare la Direzione di sottrazione: per misurare uno scavo usare UP − DOWN.
  • Verificare che i due set di punti abbiano una sovrapposizione spaziale sufficiente.
  • Assicurarsi che il Campo quota contenga quote altimetriche (Z), non coordinate planimetriche.
  • Controllare che il codice EPSG corrisponda al sistema di riferimento dei punti.
  • Verificare che la spunta "Considera solo valori positivi" non escluda tutti i pixel.

❌ I due DEM non si sovrappongono spazialmente

Il calcolo della differenza richiede un'area comune tra DEM UP e DEM DOWN. Verificare che le nuvole di punti abbiano effettivamente una zona di sovrapposizione e che il codice EPSG sia corretto. Se l'estensione comune è troppo piccola il plugin genera un errore esplicito nel log.
